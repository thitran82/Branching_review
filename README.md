# Branching Review

A two-branch literature scan on [OpenAlex](https://openalex.org). Given a seed
paper and a keyword phrase, it runs two searches inside a journal set you
control and shows where they converge:

- **Depth branch** — papers that *cite the seed*, within your journals (the
  genealogical lineage).
- **Breadth branch** — papers that *match the keywords*, within the same
  journals (the phenomenon, independent of citation lineage).
- **Convergent core** — the intersection: papers that both descend from the
  seed and are on topic.

Defaults reproduce the setup from the social cybersecurity review: seed
**Oh, Agrawal & Rao (2013)**, keywords **rumor misinformation**, and the
**IS Basket of 11 + ACM/IEEE + IS-adjacent** journal set. Every default is
editable in the interface.

## What you get per result group

Each group (depth-only, convergent core, breadth-only) has three views:

- **Papers** — the list, with title, authors, venue, year, citation count, and
  primary topic, plus a CSV export.
- **Topics & domains** — a free breakdown computed from OpenAlex metadata: the
  research domains, fields, and subfields the group spans, plus the most common
  topics and recurring keywords. No API key needed; this uses data OpenAlex
  already returns.
- **Synthesize** — an *optional* themes-methods-gaps synthesis generated by
  Claude. This is bring-your-own-key: the user pastes their own Anthropic API
  key, which is used only for that one request and never stored. Cost is borne
  by the user, not the site owner. See below.

## Bring-your-own-key synthesis (optional)

The "Synthesize" tab lets a user generate an LLM analysis of a paper group's
abstracts. It is entirely opt-in and costs the site owner nothing:

- The user supplies their own Anthropic key (from
  [console.anthropic.com](https://console.anthropic.com/settings/keys)).
- The key is sent with that single request to `api/synthesize.js`, used to call
  Anthropic once, and never written to disk or logged.
- Only titles and abstracts are sent (not full text), capped at 60 papers per
  run, so a synthesis costs the user a few cents.
- If you want to demonstrate it yourself, just paste your own key into the
  field; nothing about it is stored server-side.

You do **not** need to set any Anthropic key in Vercel's environment variables.
The OpenAlex key is the only server-side secret. The Anthropic key travels only
with each user's own synthesis request.

## How it works

The browser never touches OpenAlex directly. Two serverless functions hold the
API key and do the work:

- `api/resolve.js` — turns a DOI or title into an OpenAlex work, and a journal
  name into its ISSN.
- `api/search.js` — runs the two branches (paginated, journal-filtered),
  computes the intersection, and returns three lists.

## Run locally

You need Node 18+ and the Vercel CLI (`npm i -g vercel`).

```bash
npm install
cp .env.example .env        # then paste your key into .env
vercel dev                  # serves /api/* on :3000 and the app on :3000
```

Or run the frontend and functions separately: `vercel dev` for the API, and
`npm run dev` for the Vite dev server (it proxies `/api` to port 3000).

Get a free OpenAlex key at <https://openalex.org/settings/api> (30 seconds).
The free tier is $1/day, and one full review costs a fraction of a cent, so
this covers roughly a thousand reviews per day.

## Deploy to Vercel

Same path as before:

1. Push this folder to a new GitHub repo.
2. In Vercel, **Add New Project** and import the repo.
3. Under **Settings → Environment Variables**, add:
   - `OPENALEX_API_KEY` — your key
   - `OPENALEX_MAILTO` — your email (puts you in OpenAlex's faster pool)
4. Deploy. Vercel builds the Vite frontend to `dist/` and serves `api/*.js` as
   functions automatically.

## Verify the queries by hand

The exact OpenAlex calls the app makes are plain GET URLs you can open in a
browser. For the depth branch on the default seed and basket:

```
https://api.openalex.org/works?filter=cites:W2015843201,primary_location.source.issn:0276-7783|1047-7047|0742-1222|1536-9323|0960-085X|1350-1917|0268-3962|0963-8687|0167-9236|0378-7206|1471-7727,publication_year:2013-2026&per_page=100
```

Swap `cites:` for `search=rumor misinformation` (dropping the `cites` clause)
to see the breadth branch.

## Known limits and honest caveats

- **Not a PRISMA replacement.** This is a first-pass scaffold to accelerate
  screening, not a finished systematic review. Screen the results by hand.
- **Journal metadata.** OpenAlex ISSN coverage is good but not perfect; a
  journal with an unusual ISSN entry may need manual addition via the "Find"
  box.
- **Boolean keywords.** OpenAlex `search` ranks by relevance across
  title/abstract/full text rather than enforcing a strict `AND`. For a hard
  Boolean requirement, tighten the phrase or post-filter the exported CSV.
- **Result caps.** Each branch is capped at 600 results to keep responses fast;
  raise `cap` in `api/search.js` if a broad query needs more.
- **Rate limit.** A light per-IP limit (20 searches/minute) protects the shared
  daily budget. Adjust in `api/search.js`.
